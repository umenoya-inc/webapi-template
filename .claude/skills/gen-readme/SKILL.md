## 概要

プロジェクト内の情報源を読み取り、README.md を自動生成する。
README.md は完全に自動生成とし、手動編集は行わない。

## 情報源と対応セクション

### 1. プロジェクト概要

見出しとサブタイトルは固定:

```markdown
# Web API Template

Hono + TypeScript + Vite+ による Web API テンプレート。
```

その後に、プロジェクトの特筆すべき設計方針や特徴を 2〜4 文の散文で生成する。

- 情報源: `CLAUDE.md` の Overview / Design セクション、`docs/design/*.md` の冒頭
- 他のテンプレートや一般的な Web API プロジェクトと比べて **このプロジェクトならではの点** に絞る
- 技術スタックの羅列ではなく、設計思想とそれがもたらす具体的な効果を書く
- 前回の README の概要文は参考にしてよいが、現在の設計と乖離している場合は書き直す

### 2. 特徴

`docs/design/*.md` と `docs/rule/*.md` のファイルを読み取り、各ファイルの先頭の `# タイトル` を抽出する。
タイトルとファイルへの相対リンクを一覧にする。

- 設計方針（`docs/design/`）とルール（`docs/rule/`）はサブセクションで分ける
- 各項目はタイトルとリンクだけにし、説明文は載せない（詳細はリンク先に委譲）

### 3. モジュール構成

`src/**/index.ts` の JSDoc コメントから、モジュール名と責務の1行説明を抽出する。

- `@packageDocumentation` の直後の `## <モジュール名>` 行と、その次の空行後の説明文（1〜2行）を使う
- ネスト構造がわかるように、サブモジュールはインデントして表示する

### 4. スクリプト

`package.json` の `scripts` を読み取り、コマンド名と実行内容を表形式で列挙する。

### 5. セットアップ手順

以下の固定内容を出力する:

````markdown
## セットアップ

### 1. Vite+ のインストール

```bash
curl -fsSL https://vite.plus | bash
```

インストーラーは `.bashrc` と `.profile` に環境設定を自動追記します。
zsh を使っている場合は、以下を `.zshrc` に手動で追記してください。

```bash
. "$HOME/.vite-plus/env"
```

追記後、ターミナルを再起動してください。

### 2. Node.js のセットアップ

`.node-version` に記載されたバージョンをインストールします。

```bash
vp env install
vp env pin
```

### 3. 依存関係のインストール

```bash
vp install
```
````

## 出力ルール

- README.md 全体を上書きする（手動編集は想定しない）
- マークダウンのフォーマットはシンプルに保つ
- 情報源に変更があった場合はこのスキルを再実行して反映する
