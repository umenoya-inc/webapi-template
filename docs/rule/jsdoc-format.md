# JSDoc フォーマット

エクスポートされる関数の JSDoc コメントは、以下の構成に従う。

## 構成

```typescript
/**
 * <関数の目的を1行で>
 *
 * - <成功・エラーの各ケース>
 *
 * @remarks
 * <設計判断、採用した理由、注意点など>
 */
```

### 概要（先頭行）

- 関数が何をするかを1行で記述する。

### 振る舞い（本文）

- 概要行の直後に、関数の入出力に対する振る舞いを箇条書きで列挙する。
- 型やスキーマ制約（引数の型、戻り値の型、バリデーションルール等）は書かない。TypeScript の型で表現されている情報は重複させない。
- 成功ケースを先に、エラーケースを後に並べる。
- `defineContract` を使う関数の場合、Desc ラベル（`okAs` / `failAs` の第1引数）から導出できる。

### `@remarks`（任意）

- ソースコードやテストからは読み取れない設計判断・背景・注意点を記述する。
- このセクションは手動で記述する。自動生成の対象外。
- 記述する内容がなければ省略して良い。

## 対象

- `index.ts`（barrel export）やテーブル定義など、ロジックを持たないファイルはこのルールの対象外とする。
