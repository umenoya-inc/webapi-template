# 振る舞い分解パターンの応用

mockContract の本質は「union 型の戻り値を振る舞い名で分解して扱う」こと。モック以外にも応用できる可能性がある。

## 応用候補

- **ルートハンドラでのレスポンスマッピング** — 振る舞い名 → HTTP ステータスの宣言的対応付け
- **ログ / 観測** — 振る舞いパスごとの自動計測
- **ドキュメント生成の強化** — コード上で明示的な振る舞い名からの抽出

## 背景

defineContract + env パターンで関数の戻り値が union 型として統一されているため、振る舞いバリエーションに名前を付けて自動化する基盤がある。
