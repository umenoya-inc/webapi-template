# specdrift 対話型 update モード

## 概要

`specdrift update` に対話モードを追加し、DRIFT が検出されたアノテーションを1件ずつ確認させる。各 DRIFT に対して「update / 理由を記録してスキップ / 仕様を編集 / スキップ」を選択させることで、無言 update を構造的に防ぐ。

## 応用候補

- 理由のログ蓄積から「常にスキップされるアノテーション」を検出し、アノテーション粒度の見直しフィードバックに使う
- CI での DRIFT 検出時に理由の記録を必須化するワークフロー

## 背景

仕様書ベース開発で specdrift を運用する場合、最大のリスクは「DRIFT を確認せず update で消す」こと。`git add -p` のように1件ずつ判断を強制するインタラクションで、仕様とコードの整合性を維持する習慣を構造化する。実装コストは stdin のインタラクティブ入力のみで軽量。
