# 型の展開コスト

<!-- source: lint/type-complexity/typeExpansionCost.js@5e6dd609 -->

TypeScript の型チェッカーが型を表示する際の「展開コスト」をヒューリスティックに定量化し、型の複雑さを lint で検出する。関数における循環的複雑度に相当する指標を型に適用するアプローチ。

## 背景

LSP の hover 表示は、型チェッカーが型を再帰的に展開した結果である。型が複雑になると:

- hover が truncate されて `...` が表示される
- intersection の内部構造（`& Brand<...>`）が露出する
- AI コーディングツールのトークン消費が増大する

これらは型チェッカーが展開に「苦労している」シグナルだが、従来は目視（check-hover スキル）でしか検出できなかった。展開コストを定量化することで、lint による自動検出が可能になる。

## コスト算出アルゴリズム

型注釈の AST を葉から積み上げ、スカラー値を算出する。

| 型構造                                | コスト                                         |
| ------------------------------------- | ---------------------------------------------- |
| プリミティブ（`string`, `number` 等） | 1                                              |
| 名前付き型参照（`TSTypeReference`）   | 1 + 型引数のコスト合算                         |
| intersection / union                  | 各メンバーのコスト合算                         |
| オブジェクトリテラル（`{ ... }`）     | 各プロパティの型コスト合算（最低 1）           |
| conditional type                      | checkType + extendsType + trueType + falseType |
| mapped type（`{ [K in ...]: ... }`）  | 5（固定）+ 値型のコスト                        |
| 配列                                  | 1 + 要素型のコスト                             |
| タプル                                | 各要素のコスト合算                             |
| 関数型                                | パラメータ型 + 戻り値型のコスト合算            |
| `keyof` / `readonly` 等の演算子       | 1 + 対象型のコスト                             |
| indexed access（`T[K]`）              | objectType + indexType のコスト合算            |

### 設計判断

- **`interface` への参照はコスト 1**: hover で展開が止まるため、実質的にコスト境界として機能する。`type` alias への参照も AST 上は `TSTypeReference` で同じコスト 1 になるが、これは AST ベースの限界として許容している
- **mapped type は固定コスト 5**: キー数が静的に決定できないため、「プロパティ数件分のオブジェクト相当」として近似
- **TypeScript 型チェッカーの完全な再現は目指さない**: あくまでリファクタリングの判断材料としてのヒューリスティック。型チェッカーの展開アルゴリズムが大きく変わったタイミングで見直す程度で十分

## チェック対象

- `export` された `type` alias の定義
- `export` された関数の戻り値型注釈
- `export` された変数の型注釈

`interface` の定義は hover で展開が止まるためチェック対象外。

## 閾値

デフォルト閾値は **20**。

既存コードベースでの検証結果:

| 型               | コスト | 備考                                                                |
| ---------------- | ------ | ------------------------------------------------------------------- |
| `FlattenService` | 24     | 再帰的に service をフラット化する型。将来のリスク要因として認識済み |
| `DeriveContext`  | 22     | service の依存から context を導出する型。同上                       |
| その他すべての型 | ≤ 20   | 問題なし                                                            |

閾値 20 は「現時点で問題なく動いている型はスルーし、将来リスクのある型だけを検出する」水準として設定した。

## lint ルールとの対応

`type-complexity/type-expansion-cost` ルール（warn レベル）として実装。`npx vp check` で自動検出される。

## check-hover スキルとの棲み分け

| 検出方法               | 役割                                                                                                         |
| ---------------------- | ------------------------------------------------------------------------------------------------------------ |
| **lint ルール**        | 定量的検出。コストが閾値を超えた型を自動で警告する                                                           |
| **check-hover スキル** | 定性的検査。hover の truncation パターン、内部構造の露出、二重展開など lint では拾えない問題を目視で確認する |

lint ルールはコスト超過の「事実」を伝えるが、「どう直すべきか」の判断は check-hover の方が適している。
